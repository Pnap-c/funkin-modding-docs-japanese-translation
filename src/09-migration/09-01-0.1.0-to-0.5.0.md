# v0.1.0 から v0.5.0 への移行

## JSON マージファイルの再作成

v0.5.0 では、JSON ファイルへのマージシステムが根本的に再設計されました。以前このシステムを使用していたすべての MOD は、正常に動作させるためにこれらのファイルを再作成する必要があります。

MOD の `_merge` フォルダ内で、`json` ファイルを探し、その内容を書き換えてください。

```jsonc
// This is the format used by older versions of the game.
{
    "merge": [
        // Set `data.difficulty` to "super_hard"
        {
	        "target": "data.difficulty",
	        "payload": "super_hard"
	    },
        // In the second element of the `data.nested.enemies` array, set `weapon` to "minigun"
	    {
	        "target": "data.nested.enemies[1].weapon",
	        "payload": "minigun"
	    }
    ]
}
```

```jsonc
// This is the format which will be used starting with v0.5.0.
[
  { "op": "replace", "path": "/playData/characters/opponent", "value": "monster" }, // Replace the value of opponent with monster.
  { "op": "add", "path": "/playData/characters/girlfriend", "value": "nene" }, // Add a new key girlfriend with the value Nene.
  { "op": "add", "path": "/playData/difficulties/1", "value": "funky" }, // Add a new value funky to the difficulty array, after easy
  { "op": "add", "path": "/playData/difficulties/-", "value": "expert" }, // Add a new value expert to the end of the difficulty array.
  { "op": "remove", "path": "/playData/garbageValue" }, // Remove the key garbageValue from the data entirely
  { "op": "test", "path": "/playData/garbageValue", "value": 37 } // Test that a given value is in the JSON. If this operation fails, the patches will be rejected.
]
```

この新システムの詳細については、[ファイルの結合](../10-appending-and-merging-files/10-02-merging-files.md) をご覧ください。

## Flixel UI の廃止

[Flixel UI](https://github.com/haxeflixel/flixel-ui) は、HaxeFlixel における UI 要素の作成と UI イベントの管理に使用されるライブラリです。過去にはチャートエディタのUI構築に利用されていましたが、開発チームは本ライブラリの使い勝手の悪さを頻繁に指摘し、最終的にほとんどのユーザーインターフェースで[HaxeUI](https://github.com/haxeui)への移行を完了しました。

Friday Night Funkin' v0.5.0 では、ゲーム内でこのライブラリを使用していた最後の箇所がリファクタリングされ、ユーザーインターフェースには手動でのスプライト配置と HaxeUI の組み合わせのみが使用されるようになりました。その結果、Flixel UI は依存関係から削除されました。

Flixel UI が提供する関数やクラスを利用していたMODは、対応するリファクタリングが必要になる可能性があります。

## APIバージョンの更新

上記の移行手順を全て完了したら、最後にMODのAPIバージョン文字列を変更します。MODの`_polymod_meta.json`ファイル内で`「api_version」`プロパティを探し、`「0.5.0」`に設定してください。

```jsonc
{
    // ...

    // Change this value from "0.1.0" to "0.5.0"
    "api_version": "0.5.0",

    // ...
}
```

注記: v0.3.0 から v0.4.1 までのバージョンの Friday Night Funkin'では、MODシステムは常に `api_version` として `「0.1.0」` を検索し、それ以外の場合はMODの読み込みを拒否していました。v0.5.0アップデートにより、このバージョンはゲームの更新ごとにゲームバージョンに合わせて変更されるようになりましたが、互換性を損なう変更があった場合にのみMODが強制的に無効化されます。それらの互換性を損なう変更については、このページに記載されます。
